Chapter 7: Events

  • Event listeners-I liked the example given in the reading but I also thought another fitting example is a doorbell. We could sit at the door all day waiting for someone or something or we could listen for the doorbell or a knock to alert us and then act once it happens.
  • document.body.addEventListener("click", doSomething); is used when we want to flag a certain action instead of running an if loop over and over.
  • Event listeners are the current standard.
  • Properties screenX and screenY show the number of pixels from the top left of the screen where the event took place.
  • Properties clientX and clientY show the number of pixels from the top left of the 'client' that is being used (usually browser window).
  • Properties pageX and pageY show the number of pixels from the left and top (respectively) where the event took place in a document. This takes into account if teh page as been scrolled.
  • Mouse Events-click, mousedown, mouseup, dblclick. Do not use click and dblclick on the same element because doing so will cause both to fire.
  • Keyboard Events-keydown, keypress, and keyup. Occur in order listed but are not tied to any key in particular. keydown occurs when key is pressed and continues to occur if it is held down. keypress occurs after a keydown event but before a keyup. keypress only occurs for keys that produce character input and teh delete key. keyup occurs when a key is released.
  • Modifier Keys-shift, alt, ctrl and meta(cmd on mac) fire keydown and keyup but not a keypress event because no characters are produced on the screen.
  • Touch Events- smartphones, tablets and touch screen monitors all use touch events. touchstart occurs when user initially touches the surface. A touchstart event can interfere with usability if it fires when a user just wants to zoom, swipe, etc. click is a safer option to use because it still fires when the screen is touched but it has a 300ms delay which allows the user to do something else. touchend occurs when the user stops touching the surface. touchmove events occur when a user touches the screen and then moves around without leaving the screen. touchenter is when a user is touching the surface and then passes over an event with an event listener. touchleave occurs when the user is touching the surface but leaves the element the event listener is attatched to. touchcancel event is when a touch event is interupted like too many fingers, fingers moving off the screen, etc.
  • Touch events have properties called touches. Use touch events with caution.
  • Event listeners can be removed using removeEventListener().
  • Bubbling-when an event fires on the element clicked on first, then bubbles up the document tree, firing or 'touching' each node until it reaches the root node. Default method over capturing but both can be used.
  • Capturing-starts by firing an event on teh root element, then moves down, firing on each child element until it reaches the target element that was clicked on.
